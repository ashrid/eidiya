  <name>Task 1: Fix edit menu button dark mode visibility</name>
  <files>src/styles/main.css</files>
  <action>
Update the .menu-button class to use CSS custom properties that properly respond to dark mode.

Current code (lines 334-347):
```css
.menu-button {
  background: none;
  border: none;
  font-size: 1.5rem;
  cursor: pointer;
  padding: var(--spacing-xs);
  line-height: 1;
  color: var(--pico-muted-color);
  transition: color 0.2s ease;
}
```

Changes to make:
1. Change `background: none` to `background-color: transparent` (explicit)
2. Keep `border: none` but add `border-radius: var(--pico-border-radius)` for hover state consistency
3. Add explicit `color: var(--pico-color)` on hover instead of relying on var(--pico-muted-color) transition
4. The key issue is likely that the button inherits a background from browser defaults or parent elements in dark mode

Updated styling should be:
```css
.menu-button {
  background-color: transparent;
  border: none;
  border-radius: var(--pico-border-radius);
  font-size: 1.5rem;
  cursor: pointer;
  padding: var(--spacing-xs);
  line-height: 1;
  color: var(--pico-muted-color);
  transition: color 0.2s ease, background-color 0.2s ease;
}

.menu-button:hover {
  color: var(--pico-color);
  background-color: var(--pico-muted-border-color);
}
```

This ensures the button has explicit transparent background and proper hover state in both themes.
