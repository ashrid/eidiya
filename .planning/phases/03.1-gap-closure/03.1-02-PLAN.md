### The Badge Timing Issue
Looking at _saveEdit (lines 524-551):
```javascript
_saveEdit(field, value) {
  // ... validation ...

  // Show success feedback BEFORE dispatching to store
  this._showStatus('Saved', 'success');

  // Dispatch to store (triggers re-render, but status is already showing)
  this._onUpdate(this._contributor.id, updates);

  // Exit edit mode and re-render field
  this._exitEditMode();
  this._renderField(field);
}
```

The problem: _renderField replaces the field content, but the status-container is a sibling element (appended to the card, not inside the field). However, the store update triggers a full AppContainer re-render which destroys the entire card element, including the status badge.

The fix: The status badge needs to be shown in a way that survives the re-render, or the re-render needs to be delayed until after the badge is visible.

Options:
1. Delay the store update until after badge is shown (setTimeout)
2. Show badge in a global notification area instead of per-card
3. Prevent re-render from destroying the badge (complex)

Best approach: Option 1 - use setTimeout to delay the store update, allowing the badge to be visible before the re-render happens.

Current _saveEdit method (lines 524-551):
```javascript
_saveEdit(field, value) {
  // Validate
  const validation = this._validateField(field, value);
  if (!validation.valid) {
    this._showFieldError(field, validation.error);
    // Stay in edit mode, re-focus input
    const input = this._element.querySelector('.inline-edit-input');
    if (input) input.focus();
    return;
  }

  // Build update object
  const updates = this._buildUpdateObject(field, value);

  // Update local contributor data
  this._contributor = { ...this._contributor, ...updates };

  // Show success feedback BEFORE dispatching to store
  // (Store notification triggers re-render which destroys this card)
  this._showStatus('Saved', 'success');

  // Dispatch to store (triggers re-render, but status is already showing)
  this._onUpdate(this._contributor.id, updates);

  // Exit edit mode and re-render field
  this._exitEditMode();
  this._renderField(field);
}
```

The issue is that `this._onUpdate()` triggers a store update, which triggers AppContainer re-render, which destroys all cards including this one with its status badge.

Fix by delaying the store update:
```javascript
_saveEdit(field, value) {
  // Validate
  const validation = this._validateField(field, value);
  if (!validation.valid) {
    this._showFieldError(field, validation.error);
    // Stay in edit mode, re-focus input
    const input = this._element.querySelector('.inline-edit-input');
    if (input) input.focus();
    return;
  }

  // Build update object
  const updates = this._buildUpdateObject(field, value);

  // Update local contributor data
  this._contributor = { ...this._contributor, ...updates };

  // Show success feedback
  this._showStatus('Updated', 'success');

  // Exit edit mode and re-render field immediately
  this._exitEditMode();
  this._renderField(field);

  // Dispatch to store after a delay to allow badge to be seen
  // The store update will trigger re-render, but user will have seen the feedback
  setTimeout(() => {
    this._onUpdate(this._contributor.id, updates);
  }, 500);
}
```
