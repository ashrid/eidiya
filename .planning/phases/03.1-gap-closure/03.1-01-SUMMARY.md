# Phase 03.1 Plan 01: CSS Gap Closure Summary

**Objective:** Fix CSS-related UI/UX gaps: edit menu button visibility in dark mode, denomination input spinner overlap, and Add Contributor form width.

**Status:** COMPLETE

---

## Changes Made

### 1. Edit Menu Button Dark Mode Visibility

**Problem:** Button rendered with bright white background in dark mode, text invisible.

**Fix:** Updated `.menu-button` class (lines 334-347):
- Changed `background: none` to `background-color: transparent` (explicit)
- Added `border-radius: var(--pico-border-radius)` for hover state consistency
- Added `background-color` to transition properties
- Added explicit hover state with `var(--pico-color)` and `var(--pico-muted-border-color)`

**Commit:** `a736da1`

### 2. Denomination Input Spinner Overlap

**Problem:** Browser number input spinners overlapped the value text.

**Fix:** Added `padding-right: 1.5rem` to:
- `.denomination-field input` (line 224)
- `.breakdown-edit-field input` (line 1072)

This preserves spinner functionality while preventing overlap.

**Commit:** `93b9bad`

### 3. Add Contributor Form Width

**Problem:** Form appeared compressed in sidebar layout.

**Fix:**
- Increased `--app-max-width` from `1200px` to `1400px` (line 27)
- Increased sidebar grid column from `320px` to `360px` (line 638)

This provides 200px more total width and 40px more sidebar space.

**Commit:** `1f2ee38`
