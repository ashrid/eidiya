# Phase 3.1: Gap Closure - Context

**Gathered:** 2026-03-13
**Status:** Ready for planning
**Source:** Phase 3 Verification Gap Closure

<domain>
## Phase Boundary

This gap closure phase addresses UI/UX issues identified during Phase 3 (Display & Edit) verification.

**What this phase delivers:**
- Fix edit menu button visibility in dark mode
- Add Save/Cancel buttons to all edit modes
- Fix status badges not displaying after edits
- Fix denomination input spinner styling
- Improve Add Contributor form width

**What is NOT in scope:**
- New features beyond fixing identified gaps
- Refactoring of working functionality
- Changes to data layer or storage

</domain>

<decisions>
## Implementation Decisions

### UI/UX Fixes (Locked from verification findings)
- Edit menu button must be visible in both light and dark modes with proper contrast
- All edit modes (name, amount, denomination) must have explicit Save/Cancel buttons
- Status badges ("Updated", "Deleted") must appear after edit/delete actions and auto-dismiss
- Denomination input spinners must not overlap value text
- Add Contributor form needs adequate width for comfortable input

### Technical Approach
- Extend existing contributor card styling rather than redesign
- Use CSS custom properties for dark mode button theming
- Implement badge notification system with auto-dismiss timer
- Add padding/margin fixes for input fields to prevent spinner overlap
- Adjust sidebar/grid layout for better form width

### Claude's Discretion
- Exact button styling and placement within edit modes
- Badge animation details (fade, slide, etc.)
- Specific CSS technique for spinner styling fix
- Layout adjustment approach (flex, grid, or width changes)

</decisions>

<specifics>
## Specific Requirements from Verification

**Source:** `.planning/phases/03-display-edit/03-VERIFICATION.md`
**UAT Report:** `.planning/phases/03-display-edit/03-UAT.md`

### Gap 1: Edit Menu Button Visibility (Major)
- **Truth:** Edit menu button should be visible in both light and dark modes
- **Issue:** Button renders with bright white background in dark mode, text invisible
- **Location:** ContributorCard.js edit menu trigger

### Gap 2: Missing Save/Cancel Buttons (Major)
- **Truth:** Edit mode should have explicit Save/Cancel buttons
- **Issue:** No explicit buttons to confirm/cancel edits - relies on blur/Enter
- **Locations:** Name edit mode, amount edit mode, denomination edit mode

### Gap 3: Status Badges Not Displaying (Major)
- **Truth:** Status badges should appear after edit/delete actions
- **Issue:** Updated/Deleted badges do not appear after actions
- **Expected:** Badges show "Updated" or "Deleted" and auto-dismiss after 2 seconds

### Gap 4: Denomination Input Spinners (Minor)
- **Truth:** Denomination input spinners should not cover value text
- **Issue:** Browser number input spinners overlap the value
- **Location:** Denomination breakdown grid inputs

### Gap 5: Add Contributor Form Width (Minor)
- **Truth:** Add Contributor form should have adequate width for comfortable input
- **Issue:** Form appears compressed in sidebar layout
- **Location:** App layout / sidebar configuration

</specifics>
